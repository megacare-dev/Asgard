# Competitor Analysis

## When to use
- Quarterly competitive review
- When a new competitor emerges
- Before pricing decisions
- Before investor meetings

## Steps

1. **Read existing analysis**:
   - `docs/strategy/competitor-analysis.md` — Current landscape

2. **Research updates** for each competitor:
   - Check their GitHub repos for recent activity (stars, releases, features)
   - Check pricing page changes
   - Read their blog/changelog for new features
   - Check social media / HackerNews sentiment

3. **Update the competitor matrix** — For each competitor, evaluate:
   | Dimension | What to check |
   |:--|:--|
   | Features | New capabilities added since last review |
   | Pricing | Price changes, new tiers |
   | Community | GitHub stars growth, Discord/Slack size |
   | Enterprise | New enterprise customers, case studies |
   | Weaknesses | User complaints, missing features |

4. **Identify threats and opportunities**:
   - Are any competitors moving into our niche (self-hosted + local inference)?
   - Are there gaps they've filled that we haven't?
   - New market segments opening up?

5. **Update positioning**:
   - Does our USP still hold?
   - Update positioning statement if needed
   - Identify new differentiators to build

6. **Output**:
   - Update `docs/strategy/competitor-analysis.md`
   - Create action items for product direction if needed
   - Flag urgent threats for immediate attention

## Key competitors to track
- **Dify** (primary) — ⭐60K+ stars, LLM App Builder
- **Open WebUI** — ⭐80K+ stars, Chat UI
- **AnythingLLM** — All-in-one local AI
- **Flowise** — LLM Workflow Builder
- **LangFlow** — Visual AI Workflow
- **n8n** — Workflow Automation + AI
- **ADK-Rust** — Rust agent framework (reference, not competitor)

## Key principles
- Focus on what matters to OUR target customers, not all features
- Competitors' weaknesses = our opportunities
- Don't copy competitors — differentiate
