# Go-to-Market Strategy

## When to use
- Preparing for Community Edition launch
- Planning Enterprise Edition launch
- Entering a new market segment
- Quarterly GTM review

## Steps

1. **Define launch phases**:
   ```
   Phase 1: Stealth → Build core product (CURRENT)
   Phase 2: Developer Preview → GitHub + docs + early adopters
   Phase 3: Community Launch → v1.0, Product Hunt, HackerNews
   Phase 4: Enterprise Pilot → Design partners in target segments
   Phase 5: Enterprise GA → Sales team, pricing, support
   ```

2. **Community building plan**:
   - GitHub presence: README quality, contributing guide, issues/discussions
   - Documentation site at asgardai.dev
   - Content marketing: blog posts, tutorials, comparison guides
   - Social: Twitter/X, LinkedIn, dev communities (Reddit, HN)
   - Conference talks: local meetups → regional conferences

3. **Developer adoption strategy**:
   - One-command install: `curl -sSL https://asgardai.dev/install | bash`
   - Docker Compose quickstart (< 15 min to first query)
   - Example use cases with sample data
   - SDK/API documentation with code examples
   - Community Discord/Slack for support

4. **Enterprise sales strategy**:
   - Identify design partners in Tier 1 segments (Healthcare, Legal, Financial, Game)
   - Free Community → paid POC → Enterprise license
   - Build case studies from design partners
   - Partner with system integrators in target markets

5. **Write GTM plan** in `docs/business/go-to-market.md`:
   ```
   # Go-to-Market Strategy
   ## 1. Market Segmentation & Targeting
   ## 2. Launch Timeline & Milestones
   ## 3. Community Building Plan
   ## 4. Developer Adoption Funnel
   ## 5. Enterprise Sales Process
   ## 6. Marketing Channels & Budget
   ## 7. Metrics & KPIs
   ```

## Key principles
- Community-led growth first, enterprise sales second
- Document everything — great docs are the best marketing
- "Make it easy to start, hard to leave" — frictionless onboarding
- SEA market first, then expand globally
