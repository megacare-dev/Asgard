# Investor Pitch Deck

> "Tell a story, not a feature list."

## Purpose

Create compelling pitch materials that present Asgard's vision, traction, market opportunity, and ask to potential investors.

---

## Prerequisites — Read First

| File | Why |
|------|-----|
| `docs/README.md` | Platform overview |
| `docs/strategy/platform-review.md` | Strengths and roadmap |
| `docs/strategy/competitor-analysis.md` | Market positioning |
| `COMMERCIAL.md` | Enterprise licensing model |

---

## Process

### 1️⃣ Slide Structure (12 slides max)

| # | Slide | Content |
|---|-------|---------|
| 1 | **Title** | Asgard — Self-Hosted AI Platform · asgardai.dev |
| 2 | **Problem** | Enterprise data can't go to cloud (PDPA/HIPAA), no full-stack self-hosted AI exists |
| 3 | **Solution** | Full-stack: Gateway + RAG + Agent + Computer Use, 100% on-premise |
| 4 | **Demo** | Dashboard, RAG Pipeline, Agent Builder, Playground screenshots |
| 5 | **Market Size** | TAM/SAM/SOM with bottom-up calculation |
| 6 | **Traction** | 8 sprints done, production components, community metrics |
| 7 | **Competition** | Quadrant chart from competitor-analysis.md |
| 8 | **Business Model** | Community (AGPL) → Enterprise License → Support/SLA |
| 9 | **GTM** | Open source → design partners → enterprise |
| 10 | **Team** | Founders, key engineers, advisors |
| 11 | **Roadmap** | Phase 1 ✅ → Phase 2 🚧 → Phase 3 💰 |
| 12 | **Ask** | Funding amount, use of proceeds, milestones |

### 2️⃣ For Each Slide
- Write talking points (30-second script)
- Note which existing doc to reference
- Identify visuals needed (mermaid diagrams, screenshots)

### 3️⃣ Supporting Materials
- One-pager / executive summary
- Financial model spreadsheet reference
- Technical architecture diagram (from `docs/architecture.md`)

### 4️⃣ Output
Write to `docs/business/pitch-deck.md`

---

## Key Principles
- 1 idea per slide — no walls of text
- Lead with the problem, not the technology
- Show traction prominently — it de-risks everything
- Be specific: "$X raised at $Y valuation for Z milestones"
- Practice the "Mom test" — would a non-technical person understand?

## When to Use
Before investor meetings, accelerator applications, or when creating fundraising materials.
