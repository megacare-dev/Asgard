# Competitor Analysis

> "Know your enemy and know yourself, and you can fight a hundred battles." — Sun Tzu

## Purpose

Maintain an up-to-date competitive landscape analysis. Identify threats, opportunities, and ensure Asgard's positioning remains differentiated.

---

## Prerequisites — Read First

| File | Why |
|------|-----|
| `docs/strategy/competitor-analysis.md` | Current analysis to update |
| `docs/strategy/platform-review.md` | Our own capabilities |

---

## Process

### 1️⃣ Update Each Competitor
For each competitor (Dify, Open WebUI, AnythingLLM, Flowise, LangFlow, n8n):
- Check GitHub: stars (growth rate), recent releases, new features
- Check pricing page for changes
- Read their changelog/blog for announcements
- Check community sentiment (Reddit, HN, Twitter)

### 2️⃣ Evaluate Changes
| Dimension | What to assess |
|-----------|---------------|
| **Features** | New capabilities since last review |
| **Pricing** | Price changes, new tiers |
| **Community** | Stars growth, Discord/Slack size |
| **Enterprise** | New customers, case studies |
| **Weaknesses** | User complaints, missing features |

### 3️⃣ Threat Assessment
- Is anyone moving into our niche (self-hosted + local inference)?
- Are there gaps they've filled that we haven't?
- New entrants to track?
- Emerging patterns (A2A, MCP adoption)?

### 4️⃣ Positioning Check
- Does our USP still hold?
- Update positioning statement if needed
- Identify new differentiators to build

### 5️⃣ Output
- Update `docs/strategy/competitor-analysis.md`
- Flag urgent threats to address
- Create action items for product direction if needed
